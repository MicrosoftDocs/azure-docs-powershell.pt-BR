# Set-AzSynapseTrigger

## Sinopse
Cria um gatilho em um espaço de trabalho.

## SYNTAX

### SetByName (padrão)
```
Set-AzSynapseTrigger -WorkspaceName <String> -Name <String> -DefinitionFile <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### SetByObject
```
Set-AzSynapseTrigger -WorkspaceObject <PSSynapseWorkspace> -Name <String> -DefinitionFile <String> [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRITIVO
O cmdlet **set-AzSynapseTrigger** cria um gatilho em um espaço de trabalho. Os gatilhos são criados no estado "interrompido", o que significa que eles não começam a invocar imediatamente os pipelines referenciados.

## EXEMPLOS

### Exemplo 1
```powershell
PS C:\> Set-AzSynapseTrigger -WorkspaceName ContosoWorkspace -Name ContosoTrigger -DefinitionFile ".\scheduledTrigger.json"
```

Crie um novo gatilho chamado ContosoTrigger na ContosoWorkspace do espaço de trabalho. O gatilho é criado no estado "parado", o que significa que ele não é iniciado imediatamente. Um gatilho pode ser iniciado usando o `Start-AzDataFactoryV2Trigger` cmdlet.

### Exemplo 2
```powershell
PS C:\> $ws = Get-AzSynapseWorkspace -Name ContosoWorkspace
PS C:\> $ws | Set-AzSynapseTrigger -Name ContosoTrigger -DefinitionFile ".\scheduledTrigger.json"
```

Crie um novo gatilho chamado ContosoTrigger no espaço de trabalho ContosoWorkspace pelo pipeline. O gatilho é criado no estado "parado", o que significa que ele não é iniciado imediatamente. Um gatilho pode ser iniciado usando o `Start-AzDataFactoryV2Trigger` cmdlet.
