# Start-AzDataFactoryV2DataFlowDebugSession

## Sinopse
Inicia uma sessão de depuração do fluxo de dados no Azure data Factory

## SYNTAX

### ByFactoryName (padrão)
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-ResourceGroupName] <String> [-DataFactoryName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByFactoryObject
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob]
 [-DataFactory] <PSDataFactory> [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ByResourceId
```
Start-AzDataFactoryV2DataFlowDebugSession [[-IntegrationRuntimeFile] <String>] [-AsJob] [-ResourceId] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRITIVO
Esse comando de execução longa inicia uma sessão de depuração de fluxo de dados para os comandos de depuração futuros. Esse comando pode ser anexado a uma definição de tempo de execução de integração para configurar o tamanho/tipo do cluster de sessão de depuração.
A sequência de comandos do PowerShell para o fluxo de trabalho de depuração do fluxo de dados deve ser:
1. Start-AzDataFactoryV2DataFlowDebugSession
1. Add-AzDataFactoryV2DataFlowDebugSessionPackage
1. Invoke-AzDataFactoryV2DataFlowDebugSessionCommand (repita esta etapa para diferentes comandos/destinos ou repita a etapa 2-3 para alterar o arquivo de pacote)
1. Stop-AzDataFactoryV2DataFlowDebugSession
