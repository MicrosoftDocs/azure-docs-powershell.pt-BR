# Get-AzDenyAssignment

## Sinopse
Lista as atribuições de negação do Azure RBAC no escopo especificado.
Por padrão, ele lista todas as atribuições de negação na assinatura do Azure selecionada.
Use os parâmetros respectivos para listar as atribuições para um usuário específico ou para listar as atribuições em um recurso ou grupo de recursos específico.

## DESCRITIVO
Use o comando Get-AzDenyAssignment para listar todas as atribuições de negação que são efetivas em um escopo.
Sem parâmetros, esse comando retorna todas as atribuições Deny feitas sob a assinatura.
Essa lista pode ser filtrada usando parâmetros de filtragem para a entidade de segurança, negar o nome e o escopo da atribuição.
Para especificar um usuário, use SignInName ou os parâmetros do ObjectId do Azure AD.
Para especificar um grupo de segurança, use o parâmetro ObjectId do Azure AD.
E para especificar um aplicativo do Azure AD, use os parâmetros servicePrincipalName ou ObjectId.
O escopo no qual o acesso é negado pode ser especificado.
O padrão é a assinatura selecionada.
O escopo da atribuição Deny pode ser especificado usando-se uma das combinações de parâmetro a seguir.
Scope-este é o escopo totalmente qualificado começando com/subscriptions/ \< SubscriptionId \> .
Isso filtrará as atribuições de negação que são efetivas nesse escopo específico, ou seja, todas as atribuições de negação nesse escopo e acima.
b.
ResourceGroupName-nome de qualquer grupo de recursos na assinatura.
Isso filtrará as tarefas de forma eficaz no grupo de recursos especificado, ou seja, todas as atribuições de negação nesse escopo e acima.
partição.
ResourceFile, ResourceType, ResourceGroupName e (opcionalmente) ParentResource-identifica um recurso específico na assinatura e filtra as atribuições de negação efetivas nesse escopo de recursos.
Para determinar o que o acesso é negado para um usuário específico na assinatura, use a opção ExpandPrincipalGroups
Isso listará todas as atribuições de negação atribuídas ao usuário e os grupos dos quais o usuário é membro.
