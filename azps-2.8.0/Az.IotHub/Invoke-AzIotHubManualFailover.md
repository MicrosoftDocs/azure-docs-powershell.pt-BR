# Invoke-AzIotHubManualFailover

## Sinopse
Invocar o processo de failover para o Hub IoT para a região de recuperação de desastres em par geográfico.

## SYNTAX

### ResourceSet (padrão)
```
Invoke-AzIotHubManualFailover [-ResourceGroupName] <String> [-Name] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
Invoke-AzIotHubManualFailover [-InputObject] <PSIotHub> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceSet
```
Invoke-AzIotHubManualFailover [-ResourceId] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRITIVO
Ele disparará o failover do seu hub IoT para o local secundário. Esta ação causará perda de tempo e telemetria para a sua solução. Esta é uma operação de longa duração que pode demorar vários minutos para ser concluída. Por favor, tome cuidado ao usá-lo.

## EXEMPLOS

### Exemplo 1
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub"
```

Iniciando o processo de failover do Hub IoT "myiothub".

### Exemplo 2
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub" -AsJob
```

Iniciando o processo de failover do Hub IoT "myiothub" em segundo plano.
