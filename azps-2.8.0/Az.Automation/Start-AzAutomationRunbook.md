# Start-AzAutomationRunbook

## Sinopse
Inicia um trabalho de runbook.

## SYNTAX

### ByAsynchronousReturnJob (padrão)
```
Start-AzAutomationRunbook [-Name] <String> [-Parameters <IDictionary>] [-RunOn <String>]
 [-ResourceGroupName] <String> [-AutomationAccountName] <String> [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

### BySynchronousReturnJobOutput
```
Start-AzAutomationRunbook [-Name] <String> [-Parameters <IDictionary>] [-RunOn <String>] [-Wait]
 [-MaxWaitSeconds <Int32>] [-ResourceGroupName] <String> [-AutomationAccountName] <String>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## DESCRITIVO
O cmdlet **Start-AzAutomationRunbook** inicia um trabalho do runbook de automação do Azure.
Especifique a ID ou o nome de um runbook.

## EXEMPLOS

### Exemplo 1: iniciar um trabalho de runbook
```
PS C:\>Start-AzAutomationRunbook -AutomationAccountName "Contoso17" -Name "Runbk01" -ResourceGroupName "ResourceGroup01"
```

Esse comando inicia um trabalho de runbook para o runbook chamado Runbk01 na conta de automação do Azure chamada Contoso17.

### Exemplo 2: iniciar um trabalho de runbook do Python 2 com parâmetros
```
PS C:\>Start-AzAutomationRunbook -AutomationAccountName "Contoso17" -Name "RunbkPy01" -ResourceGroupName "ResourceGroup01" -Parameters @{"Key1"="ValueForPosition1";"Key2"="ValueForPosition2"}
```

Esse comando inicia um trabalho de runbook para o runbook 2 do Python chamado RunbkPy01 na conta de automação do Azure chamada Contoso17 com "ValueForPosition1" como o primeiro parâmetro posicional e "ValueForPosition2" para o segundo.

### Exemplo 3: iniciar um trabalho de runbook e esperar pelos resultados
```
Start-AzAutomationRunbook -AutomationAccountName "Contoso17" -Name "Runbk01" -ResourceGroupName "ResourceGroup01" -MaxWaitSeconds 1000 -Wait
```

Esse comando inicia um trabalho de runbook para o runbook chamado Runbk01 na conta de automação do Azure chamada Contoso17.
Esse comando especifica o parâmetro _Wait_ .
Portanto, retorna os resultados após a conclusão do trabalho.
O cmdlet aguarda até 1000 segundos para os resultados.
