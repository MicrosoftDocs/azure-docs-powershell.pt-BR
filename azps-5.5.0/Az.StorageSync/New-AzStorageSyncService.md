# New-AzStorageSyncService

## Sinopse
Esse comando cria um novo serviço de sincronização de armazenamento em um grupo de recursos.

## Sintaxe

```
New-AzStorageSyncService [-ResourceGroupName] <String> [-Name] <String> [-Location] <String> [-IncomingTrafficPolicy] <String> [-Tag <Hashtable>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Descrição
Um serviço de sincronização de armazenamento é o recurso de nível superior para a Sincronização de Arquivos do Azure. Esse comando cria um novo serviço de sincronização de armazenamento em um grupo de recursos. Recomendamos criar o menor serviço de sincronização de armazenamento necessário para diferenciar grupos distintos de servidores em sua organização. Um serviço de sincronização de armazenamento contém grupos de sincronização e também funciona como um destino para registrar seus servidores. Um servidor só pode ser registrado em um único serviço de sincronização de armazenamento. Se os servidores precisarem participar da sincronização do mesmo conjunto de arquivos, registre-os no mesmo serviço de sincronização de armazenamento.

## Exemplos

### Exemplo 1
```powershell
PS C:\> New-AzStorageSyncService -ResourceGroupName "myResourceGroup" -Location "myLocation" -StorageSyncServiceName "myStorageSyncServiceName" -IncomingTrafficPolicy "AllowAllTraffic"
```

Esse comando criará um serviço de sincronização de armazenamento.
