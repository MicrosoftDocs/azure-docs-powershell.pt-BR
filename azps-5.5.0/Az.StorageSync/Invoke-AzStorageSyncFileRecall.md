# Invoke-AzStorageSyncFileRecall

## Sinopse
Esse comando faz com que todos os arquivos em camadas voltem para o disco local.

## Sintaxe

### StringParameterSet (Padrão)
```
Invoke-AzStorageSyncFileRecall [-ResourceGroupName] <String> [-StorageSyncServiceName] <String>
 [-SyncGroupName] <String> [-Name] <String> [-Pattern <String>] [-RecallPath <String>] [-PassThru] [-AsJob]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdParameterSet
```
Invoke-AzStorageSyncFileRecall [-ResourceId] <String> [-Pattern <String>] [-RecallPath <String>] [-PassThru]
 [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ObjectParameterSet
```
Invoke-AzStorageSyncFileRecall [-InputObject] <PSServerEndpoint> [-Pattern <String>] [-RecallPath <String>]
 [-PassThru] [-AsJob] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Descrição
Quando a camada de nuvem está habilitada em um ponto de extremidade de servidor (um local específico em um servidor registrado), esse comando pode ser usado para lembrar todos os arquivos hierárquidos em disco local. É altamente recomendável desabilitar a camada de nuvem neste ponto de extremidade do servidor antes de iniciar o recall. Se o tiering ainda estiver em, o recall pode fazer com que outros arquivos possam ser hierárquidos, o que não atingirá o objetivo desejado de todo o conteúdo que reside no disco local.

## Exemplos

### Exemplo 1
```powershell
PS C:\> Invoke-AzStorageSyncFileRecall -ResourceGroupName "myResourceGroup" -StorageSyncServiceName "myStorageSyncServiceName" -SyncGroupName "mySyncGroupName" -ServerEndpointName "myServerEndpointName"
```

Esse comando lembra recursivamente todos os arquivos de camada localizados sob o caminho raiz do ponto de extremidade do servidor especificado.
