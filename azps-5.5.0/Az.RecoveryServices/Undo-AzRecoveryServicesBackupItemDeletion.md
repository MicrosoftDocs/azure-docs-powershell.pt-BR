# Undo-AzRecoveryServicesBackupItemDeletion

## Sinopse
Se um item de backup for excluído e presente em um estado de exclusão suave, esse comando retornará ao estado em que os dados serão mantidos para sempre 

## Sintaxe

```
Undo-AzRecoveryServicesBackupItemDeletion [-Item] <ItemBase> [-Force] [-VaultId <String>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## Descrição
O Undo-AzRecoveryServicesBackupItemDeletion cmdlet reverte um item excluído suavemente para um estado em que a proteção é interrompida, mas os dados são mantidos para sempre.

## Exemplos

### Exemplo 1
```powershell
PS C:\> $Cont = Get-AzRecoveryServicesBackupContainer -ContainerType AzureVM
PS C:\> $PI = Get-AzRecoveryServicesBackupItem -Container $Cont[0] -WorkloadType AzureVM 
PS C:\> Disable-AzRecoveryServicesBackupProtection -Item $PI[0] -RemoveRecoveryPoints
PS C:\> $PI = Get-AzRecoveryServicesBackupItem -Container $Cont[0] -WorkloadType AzureVM | Where-Object {$_.DeleteState -eq "ToBeDeleted"}
PS C:\> Undo-AzRecoveryServicesBackupItemDeletion -Item $PI[0]
```

O primeiro comando obtém uma matriz de contêineres de backup e, em seguida, o armazena na matriz $Cont dados.
O segundo comando obtém o item de Backup correspondente ao primeiro item de contêiner e o armazena na variável $PI dados.
O terceiro comando desabilita a proteção de backup para o item no $PI 0 e coloca o item em um \[ \] estado softdeleted.
O quarto comando obtém o item que está em um estado softdeleted.
O último comando leva o VM softdeleted a um estado onde a proteção é interrompida, mas os dados são mantidos para sempre.

### Exemplo 2

Reativa um Item excluído suavemente. (gerado automaticamente)

```powershell
<!-- Aladdin Generated Example --> 
Undo-AzRecoveryServicesBackupItemDeletion -Item $PI[0] -VaultId $vault.ID
```
