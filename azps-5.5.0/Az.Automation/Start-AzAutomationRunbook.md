# Start-AzAutomationRunbook

## Sinopse
Inicia um trabalho de livro de executar.

## Sintaxe

### ByAsynchronousReturnJob (Padrão)
```
Start-AzAutomationRunbook [-Name] <String> [-Parameters <IDictionary>] [-RunOn <String>]
 [-ResourceGroupName] <String> [-AutomationAccountName] <String> [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

### BySynchronousReturnJobOutput
```
Start-AzAutomationRunbook [-Name] <String> [-Parameters <IDictionary>] [-RunOn <String>] [-Wait]
 [-MaxWaitSeconds <Int32>] [-ResourceGroupName] <String> [-AutomationAccountName] <String>
 [-DefaultProfile <IAzureContextContainer>] [<CommonParameters>]
```

## Descrição
O **cmdlet Start-AzAutomationRunbook** inicia um trabalho de manual de automação do Azure.
Especifique a ID ou o nome de um livro de executar.

## Exemplos

### Exemplo 1: Iniciar um trabalho de uma agenda de trabalho
```
PS C:\>Start-AzAutomationRunbook -AutomationAccountName "Contoso17" -Name "Runbk01" -ResourceGroupName "ResourceGroup01"
```

Esse comando inicia um trabalho de runbook para o runbook chamado Runbk01 na conta de Automação do Azure chamada Contoso17.

### Exemplo 2: Iniciar um trabalho de uma agenda de trabalho do Python 2 com parâmetros
```
PS C:\>Start-AzAutomationRunbook -AutomationAccountName "Contoso17" -Name "RunbkPy01" -ResourceGroupName "ResourceGroup01" -Parameters @{"Key1"="ValueForPosition1";"Key2"="ValueForPosition2"}
```

Este comando inicia um trabalho de runbook para o livro de runbook Python 2 chamado RunbkPy01 na conta de Automação do Azure chamada Contoso17 com "ValueForPosition1" como o primeiro parâmetro posicional e "ValueForPosition2" para o segundo.

### Exemplo 3: Iniciar um trabalho de uma agenda de trabalho e aguardar os resultados
```
Start-AzAutomationRunbook -AutomationAccountName "Contoso17" -Name "Runbk01" -ResourceGroupName "ResourceGroup01" -MaxWaitSeconds 1000 -Wait
```

Esse comando inicia um trabalho de runbook para o runbook chamado Runbk01 na conta de Automação do Azure chamada Contoso17.
Esse comando especifica o parâmetro _Aguardar._
Portanto, ele retorna resultados depois que o trabalho é concluído.
O cmdlet espera até 1000 segundos pelos resultados.
