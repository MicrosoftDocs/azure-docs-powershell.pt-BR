# Add-AzApplicationGatewayProbeConfig

## Sinopse
Adiciona um problema de saúde a um Gateway de Aplicativo.

## Sintaxe

```
Add-AzApplicationGatewayProbeConfig -ApplicationGateway <PSApplicationGateway> -Name <String>
 -Protocol <String> [-HostName <String>] -Path <String> -Interval <Int32> -Timeout <Int32>
 -UnhealthyThreshold <Int32> [-PickHostNameFromBackendHttpSettings] [-MinServers <Int32>]
 [-Match <PSApplicationGatewayProbeHealthResponseMatch>] [-DefaultProfile <IAzureContextContainer>]
 [<CommonParameters>]
```

## Descrição
O Add-AzApplicationGatewayProbeConfig cmdlet adiciona um problema de saúde a um Gateway de Aplicativos.

## Exemplos

### Exemplo 1: Adicionar um problema de saúde a um gateway de aplicativo
```
PS C:\>$Probe = Add-AzApplicationGatewayProbeConfig -ApplicationGateway Gateway -Name "Probe01" -Protocol Http -HostName "contoso.com" -Path "/path/custompath.htm" -Interval 30 -Timeout 120 -UnhealthyThreshold 8
```

Este comando adiciona uma confirmação de saúde chamada " Sor01 " para o gateway de aplicativo chamado Gateway.
O comando também define o limite não saudável para 8 recuperações e horas após 120 segundos.
