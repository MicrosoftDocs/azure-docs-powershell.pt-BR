# Invoke-AzSqlDatabaseFailover

## Sinopse
Falha em um banco de dados.

## Sintaxe

```
Invoke-AzSqlDatabaseFailover [-DatabaseName] <String> [-ReadableSecondary] [-AsJob] [-PassThru] [-Force]
 [-ServerName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## Descrição
O Invoke-AzSqlDatabaseFailover cmdlet failovers de um banco de dados SQL do Azure. Se o banco de dados estiver em um pool elástica, esse comando fará failover no banco de dados determinado sem afetar os outros bancos de dados no mesmo pool elástica.

## Exemplos

### Exemplo 1
```powershell
PS C:\> Invoke-AzSqlDatabaseFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
```

Esse comando fará failover na réplica principal do banco de dados chamado "Database01" no servidor chamado "Server01"

### Exemplo 2
```powershell
PS C:\> Invoke-AzSqlDatabaseFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -ReadableSecondary
```

Esse comando fará failover a réplica secundária acessível do banco de dados chamado "Database01" no servidor chamado "Server01"
