# Set-AzSqlServerThreatDetectionPolicy

## Sinopse
Define uma política de detecção de ameaças em um servidor.

## SYNTAX

```
Set-AzSqlServerThreatDetectionPolicy [-PassThru] [-NotificationRecipientsEmails <String>]
 [-EmailAdmins <Boolean>] [-ExcludedDetectionType <String[]>] [-StorageAccountName <String>]
 [-RetentionInDays <UInt32>] -ServerName <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRITIVO
O cmdlet **set-AzSqlServerThreatDetectionPolicy** define uma política de detecção de ameaças em um SQL Server do Azure.
Para habilitar a detecção de ameaças em um servidor, uma política de auditoria deve estar habilitada nesse servidor.
Para usar esse cmdlet, especifique os parâmetros *ResourceGroupName* e ServerName para identificar o servidor.

## EXEMPLOS

### Exemplo 1: definir a política de detecção de ameaças para um banco de dados
```
PS C:\>Set-AzSqlServerThreatDetectionPolicy -ResourceGroupName "ResourceGroup11" -ServerName "Server01" -NotificationRecipientsEmails "admin01@contoso.com;secadmin@contoso.com" -EmailAdmins $False -ExcludedDetectionType "Sql_Injection_Vulnerability","SQL_Injection" -StorageAccountName "mystorageAccount"
```

Este comando define a política de detecção de ameaças para um servidor chamado Server01.
