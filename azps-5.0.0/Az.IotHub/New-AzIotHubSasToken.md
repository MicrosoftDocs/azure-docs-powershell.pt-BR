# New-AzIotHubSasToken

## Sinopse
Gerar um token SAS para um hub IoT, dispositivo ou módulo de destino.

## SYNTAX

### ResourceSet (padrão)
```
New-AzIotHubSasToken [-ResourceGroupName] <String> [-IotHubName] <String> [-DeviceId <String>]
 [-ModuleId <String>] [-KeyName <String>] [-KeyType <PSKeyType>] [-Duration <Int32>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
New-AzIotHubSasToken [-InputObject] <PSIotHub> [-DeviceId <String>] [-ModuleId <String>] [-KeyName <String>]
 [-KeyType <PSKeyType>] [-Duration <Int32>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

### ResourceSet
```
New-AzIotHubSasToken [-ResourceId] <String> [-DeviceId <String>] [-ModuleId <String>] [-KeyName <String>]
 [-KeyType <PSKeyType>] [-Duration <Int32>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## DESCRITIVO
Para tokens SAS de dispositivo, o parâmetro política é usado para acessar somente o registro do dispositivo. Portanto, a política deve ter acesso de leitura ao registro.
Para tokens de Hub IoT, a política faz parte da SAS.

## EXEMPLOS

### Exemplo 1
```powershell
PS C:\> New-AzIotHubSasToken -ResourceGroupName "myresourcegroup" -IotHubName "myiothub"
```

Gere um token SAS de Hub IoT usando a política iothubowner e a chave primária.

### Exemplo 2
```powershell
PS C:\> New-AzIotHubSasToken -ResourceGroupName "myresourcegroup" -IotHubName "myiothub" -KeyName "registryRead" -KeyType "secondary"
```

Gere um token SAS de Hub IoT usando a política registryRead e a chave secundária.

### Exemplo 3
```powershell
PS C:\> New-AzIotHubSasToken -ResourceGroupName "myresourcegroup" -IotHubName "myiothub" -DeviceId "myDevice1"
```

Gere um token de dispositivo SAS usando a política iothubowner para acessar o registro de dispositivo {iothub_name}.

### Exemplo 4
```powershell
PS C:\> New-AzIotHubSasToken -ResourceGroupName "myresourcegroup" -IotHubName "myiothub" -DeviceId "myDevice1" -ModuleId "myModule1"
```

Gere um token de módulo SAS usando a política iothubowner para acessar o registro de dispositivo {iothub_name}.
