# New-AzServiceFabricApplication

## Sinopse
Criar novo aplicativo Service Fabric no grupo de recursos e no cluster especificado.

## SYNTAX

### SkipAppTypeVersion (padrão)
```
New-AzServiceFabricApplication [-ResourceGroupName] <String> [-ClusterName] <String>
 [-ApplicationTypeName] <String> [-ApplicationTypeVersion] <String> -Name <String>
 [-ApplicationParameter <Hashtable>] [-MinimumNodeCount <Int64>] [-MaximumNodeCount <Int64>] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### CreateAppTypeVersion
```
New-AzServiceFabricApplication [-ResourceGroupName] <String> [-ClusterName] <String>
 [-ApplicationTypeName] <String> [-ApplicationTypeVersion] <String> -Name <String>
 [-ApplicationParameter <Hashtable>] -PackageUrl <String> [-MinimumNodeCount <Int64>]
 [-MaximumNodeCount <Int64>] [-Force] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## DESCRITIVO
Esse cmdlet cria um novo aplicativo de malha de serviços no grupo de recursos e no cluster especificado. O parâmetro-PackageUrl pode ser usado para criar a versão do tipo, se a versão do tipo já estiver sendo encerrada, mas seu estado in ' Failed ', o cmdlet perguntará se o usuário deseja recriar a versão do tipo. Se continuar no estado ' falha ', o comando irá parar o processo e gerar uma exceção.

## EXEMPLOS

### Exemplo 1
```powershell
PS C:\> New-AzServiceFabricApplication -ResourceGroupName "testRG" -ClusterName "testCluster" -ApplicationTypeName "TestAppType" -ApplicationTypeVersion "v1" -Name "testApp" -ApplicationParameter @{key0="value0";key1=$null;key2="value2"}
```

Este exemplo cria o aplicativo "testApp" em grupo de recursos "testRG" e cluster "testCluster". O tipo de aplicativo "TestAppType" versão "v1" já deve existir no cluster, e os parâmetros do aplicativo devem ser definidos no manifesto do aplicativo, caso contrário, o cmdlet irá falhar.

### Exemplo 2: especificar-PackageUrl para criar a versão do tipo de aplicativo antes de criar o aplicativo.
```powershell
PS C:\> New-AzServiceFabricApplication -ResourceGroupName "testRG" -ClusterName "testCluster" -ApplicationTypeName "TestAppType" -ApplicationTypeVersion "v1" -Name "testApp" -PackageUrl "https://sftestapp.blob.core.windows.net/sftestapp/testApp_1.0.sfpkg" -ApplicationParameter @{key0="value0";key1=$null;key2="value2"}
```

Este exemplo cria o tipo de aplicativo "TestAppType" versão "v1" usando a URL do pacote fornecida. Depois disso, ele vai continuar o processo normal para criar o aplicativo. Se a versão do tipo de aplicativo já estiver sendo encerrada e o estado de provisionamento que estava em ' Failed ', ele solicitará que ele decida se o usuário quer recriar a versão do tipo.
