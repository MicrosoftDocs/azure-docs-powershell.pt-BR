# Stop-AzTenantDeployment

## Sinopse
Cancelar uma implantação em execução no escopo do locatário

## SYNTAX

### StopByDeploymentName (padrão)
```
Stop-AzTenantDeployment [-Name] <String> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>]
 [-WhatIf] [-Confirm] [<CommonParameters>]
```

### StopByDeploymentId
```
Stop-AzTenantDeployment -Id <String> [-PassThru] [-Pre] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### StopByInputObject
```
Stop-AzTenantDeployment -InputObject <PSDeployment> [-PassThru] [-Pre]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRITIVO
O cmdlet **Stop-AzTenantDeployment** cancela uma implantação que foi iniciada, mas não foi concluída no escopo do locatário atual.
Para interromper uma implantação, a implantação deve ter um estado de provisionamento incompleto, como provisionamento, e não um estado concluído, como provisionado ou com falha.

Para criar uma implantação no escopo do locatário, use o cmdlet New-AzTenantDeployment.

## EXEMPLOS

### Exemplo 1
```
PS C:\>Stop-AzTenantDeployment -Name "deployment01"
```

Esse comando cancela uma implantação em execução "deployment01" no escopo do locatário atual.

### Exemplo 2
```
PS C:\>Get-AzTenantDeployment -Name "deployment01" | Stop-AzTenantDeployment
```

Esse comando obtém a implantação "deployment01" no escopo do locatário atual e a cancela.
