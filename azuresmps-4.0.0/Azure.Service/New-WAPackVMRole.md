# New-WAPackVMRole

## Sinopse
Cria uma função de máquina virtual.

## SYNTAX

### QuickCreate (padrão)
```
New-WAPackVMRole -Name <String> -Label <String> -ResourceDefinition <VMRoleResourceDefinition>
 [-Profile <AzureSMProfile>] [<CommonParameters>]
```

### FromCloudService
```
New-WAPackVMRole -Name <String> -Label <String> -ResourceDefinition <VMRoleResourceDefinition>
 -CloudService <CloudService> [-Profile <AzureSMProfile>] [<CommonParameters>]
```

## DESCRITIVO
Esses tópicos são preteridos e serão removidos no futuro.
Este tópico descreve o cmdlet na versão 0.8.1 do módulo do Microsoft Azure PowerShell.
Para descobrir a versão do módulo que você está usando, no console do PowerShell do Azure, digite `(Get-Module -Name Azure).Version` .

O cmdlet **New-WAPackVMRole** cria uma função de máquina virtual.

## EXEMPLOS

### Exemplo 1: criar uma função de máquina virtual (comportamento de emulação de WAP)
```
PS C:\> New-WAPackVMRole -Name "ContosoVMRole01" -Label "ContosoVMRoleLabel01" -ResourceDefinition $resdef
```

Como não especificamos qualquer serviço na nuvem (comportamento de emulação de WAP), o comando criará um serviço na nuvem para nós que terá o mesmo nome que a função da máquina virtual.
Nesse caso, o comando a seguir criará uma função de máquina virtual com o nome ContosoVMRole01, Label ContosoVMRoleLabel01.
Observe que a definição do recurso que está sendo usado aqui deve ser criada manualmente, embora o PowerShell.
