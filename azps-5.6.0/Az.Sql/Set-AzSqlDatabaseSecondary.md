# Set-AzSqlDatabaseSecondary

## SYNOPSIS
Alterna um banco de dados secundário para ser principal para iniciar o failover.

## SINTAXE

### NoOptionsSet (Padrão)
```
Set-AzSqlDatabaseSecondary [-DatabaseName] <String> -PartnerResourceGroupName <String> [-AsJob]
 [-ServerName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

### ByFailoverParams
```
Set-AzSqlDatabaseSecondary [-DatabaseName] <String> -PartnerResourceGroupName <String> [-Failover]
 [-AllowDataLoss] [-AsJob] [-ServerName] <String> [-ResourceGroupName] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
O cmdlet **Set-AzSqlDatabaseSecondary** alterna um banco de dados secundário para ser principal para iniciar o failover.
Este cmdlet foi projetado como um comando de configuração geral, mas atualmente está limitado a iniciar o failover.
Especifique *o parâmetro AllowDataLoss* para iniciar um failover de força durante uma paralisação.
Você não precisa especificar esse parâmetro quando executar uma operação planejada, como a broca de recuperação.
No último caso, o banco de dados secundário é sincronizado com o principal antes de ser comutado.

## EXEMPLOS

### Exemplo 1: Iniciar um failover planejado
```powershell
$database = Get-AzSqlDatabase -DatabaseName $databaseName -ResourceGroupName $secondaryResourceGroupName -ServerName $secondaryServerName
$database | Set-AzSqlDatabaseSecondary -PartnerResourceGroupName $primaryResourceGroupName -Failover
```

### Exemplo 2: Iniciar um failover forçado (com perda potencial de dados)
```powershell
$database = Get-AzSqlDatabase -DatabaseName $databaseName -ResourceGroupName $secondaryResourceGroupName -ServerName $secondaryServerName
$database | Set-AzSqlDatabaseSecondary -PartnerResourceGroupName $primaryResourceGroupName -Failover -AllowDataLoss
```
