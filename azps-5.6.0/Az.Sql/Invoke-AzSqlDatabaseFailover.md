# Invoke-AzSqlDatabaseFailover

## SYNOPSIS
Failovers um banco de dados.

## SINTAXE

```
Invoke-AzSqlDatabaseFailover [-DatabaseName] <String> [-ReadableSecondary] [-AsJob] [-PassThru] [-Force]
 [-ServerName] <String> [-ResourceGroupName] <String> [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
O Invoke-AzSqlDatabaseFailover cmdlet de falha em um banco de dados do Azure SQL. Se o banco de dados estiver em um pool elástica, esse comando fará failover no banco de dados determinado sem afetar os outros bancos de dados no mesmo pool elástica.

## EXEMPLOS

### Exemplo 1
```powershell
PS C:\> Invoke-AzSqlDatabaseFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01"
```

Este comando fará failover da réplica principal do banco de dados chamado "Database01" no servidor chamado "Server01"

### Exemplo 2
```powershell
PS C:\> Invoke-AzSqlDatabaseFailover -ResourceGroupName "ResourceGroup01" -ServerName "Server01" -DatabaseName "Database01" -ReadableSecondary
```

Este comando fará failover da réplica secundária acessível do banco de dados chamado "Database01" no servidor chamado "Server01"
