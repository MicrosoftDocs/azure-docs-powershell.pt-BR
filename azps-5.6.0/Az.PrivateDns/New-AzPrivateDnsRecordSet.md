# New-AzPrivateDnsRecordSet

## SYNOPSIS
Cria um conjunto de registros em uma zona DNS privada.

## SINTAXE

### Campos (Padrão)
```
New-AzPrivateDnsRecordSet -ResourceGroupName <String> -ZoneName <String> -Name <String>
 -RecordType <RecordType> -Ttl <UInt32> [-Metadata <Hashtable>] [-PrivateDnsRecord <PSPrivateDnsRecordBase[]>]
 [-Overwrite] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### Objeto
```
New-AzPrivateDnsRecordSet -Zone <PSPrivateDnsZone> -Name <String> -RecordType <RecordType> -Ttl <UInt32>
 [-Metadata <Hashtable>] [-PrivateDnsRecord <PSPrivateDnsRecordBase[]>] [-Overwrite]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceId
```
New-AzPrivateDnsRecordSet -ParentResourceId <String> -Name <String> -RecordType <RecordType> -Ttl <UInt32>
 [-Metadata <Hashtable>] [-PrivateDnsRecord <PSPrivateDnsRecordBase[]>] [-Overwrite]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
O New-AzPrivateDnsRecordSet cmdlet cria um novo registro DNS (Sistema de Nomes de Domínio Privado) definido com o nome especificado e digite na zona privada especificada. Um objeto RecordSet é um conjunto de registros DNS privados com o mesmo nome e tipo. Observe que o nome é relativo à zona privada e não ao nome totalmente qualificado. O parâmetro PrivateDnsRecord especifica os registros no conjunto de registros. Esse parâmetro usa uma matriz de registros DNS privados, construídos usando New-AzPrivateDnsRecordConfig. Você pode usar o operador de pipeline para passar um objeto PSPrivateDnsZone para esse cmdlet, ou pode passar um objeto PSPrivateDnsZone como o parâmetro Zone, ou pode especificar a zona por sua ResourceId ou, como alternativa, você pode especificar a zona pelo nome. Você pode usar o parâmetro Confirm e $ConfirmPreference Windows PowerShell para controlar se o cmdlet solicita a confirmação. Se um RecordSet correspondente já existir (mesmo nome e tipo de registro), você deverá especificar o parâmetro Overwrite, caso contrário, o cmdlet não criará um novo RecordSet .

PS C:\> $Records += New-AzPrivateDnsRecordConfig -IPv4Address 1.2.3.4

# When creating a RecordSet containing a single record, the above sequence can also be condensed into a single line:

PS C:\> $RecordSet = New-AzPrivateDnsRecordSet -Name "www" -RecordType A -ResourceGroupName "MyResourceGroup" -TTL 3600 -ZoneName "myzone.com" -PrivateDnsRecords (New-AzPrivateDnsRecordConfig -IPv4Address 1.2.3.4)

Ttl               : 3600

### Exemplo 8: Criar um RecordSet no apex de zona
```powershell
PS C:\> $Records = @()
PS C:\> $Records += New-AzPrivateDnsRecordConfig -Ipv4Address 1.2.3.4
PS C:\> $RecordSet = New-AzPrivateDnsRecordSet -Name "@" -RecordType A -ResourceGroupName "MyResourceGroup" -TTL 3600 -ZoneName "myzone.com" -PrivateDnsRecords $Records

Id                : /subscriptions/xxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx/resourceGroups/myresourcegroup/providers/Micros
                    oft.Network/privateDnsZones/myzone.com/A/@
Name              : @
ZoneName          : myzone.com
ResourceGroupName : MyResourceGroup
Ttl               : 3600
Etag              : xxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
RecordType        : A
Records           : {1.2.3.4}
Metadata          :
IsAutoRegistered  :
```

Este comando cria um RecordSet no véreo (ou raiz) da zona privada myzone.com. Para fazer isso, o nome do conjunto de registros é especificado como "@" (incluindo aspas duplas). Não é possível criar registros CNAME no vérme de uma zona. Essa é uma restrição dos padrões DNS; não é uma limitação do DNS privado do Azure. Para criar um RecordSet usando apenas uma linha de pn_PowerShell_short, ou para criar um conjunto de registros com vários registros, consulte o Exemplo 1.

### Exemplo 11: Criar um conjunto de registros e suprimir toda a confirmação

```powershell
PS C:\>$RecordSet = New-AzPrivateDnsRecordSet -Name "www" -RecordType A -ResourceGroupName "MyResourceGroup" -TTL 3600 -ZoneName "myzone.com" -PrivateDnsRecords (New-AzDnsRecordConfig -Ipv4Address 1.2.3.4) -Confirm:$False -Overwrite
```

Este comando cria um RecordSet. O parâmetro Overwrite garante que esse conjunto de registros sobrescreva qualquer conjunto de registros pré-existentes com o mesmo nome e tipo (os registros existentes nesse conjunto de registros são perdidos). O parâmetro Confirm com um valor de $False suprime o prompt de confirmação.
