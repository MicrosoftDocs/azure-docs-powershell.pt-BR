# Restore-AzKeyVaultManagedStorageAccount

## SYNOPSIS
Restaura uma conta de armazenamento gerenciado em um cofre de chaves de um arquivo de backup.

## SINTAXE

### ByVaultName (Padrão)
```
Restore-AzKeyVaultManagedStorageAccount [-VaultName] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Restore-AzKeyVaultManagedStorageAccount [-InputObject] <PSKeyVault> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Restore-AzKeyVaultManagedStorageAccount [-ResourceId] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
O cmdlet **Restore-AzKeyVaultManagedStorageAccount** cria uma conta de armazenamento gerenciado no cofre de chaves especificado de um arquivo de backup.
Essa conta de armazenamento gerenciado é uma réplica da conta de armazenamento gerenciado com backup no arquivo de entrada e tem o mesmo nome do original.
Se o cofre de chaves já contiver uma conta de armazenamento gerenciada pelo mesmo nome, esse cmdlet falhará em vez de sobrescrever o original.
O cofre de chaves em que você restaura a conta de armazenamento gerenciado pode ser diferente do cofre de chaves do que você fez o backup da conta de armazenamento gerenciado.
No entanto, o cofre de chaves deve usar a mesma assinatura e estar em uma região do Azure na mesma geografia (por exemplo, América do Norte).
Consulte o Centro de Confiança do Microsoft Azure ( https://azure.microsoft.com/support/trust-center/) para o mapeamento de regiões do Azure para geografias.

## EXEMPLOS

### Exemplo 1: Restaurar uma conta de armazenamento gerenciado com backup
```powershell
PS C:\> Restore-AzKeyVaultManagedStorageAccount -VaultName 'MyKeyVault' -InputFile "C:\Backup.blob"
```
