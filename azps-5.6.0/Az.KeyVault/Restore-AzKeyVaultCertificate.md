# Restore-AzKeyVaultCertificate

## SYNOPSIS
Restaura um certificado em um cofre de chaves de um arquivo de backup.

## SINTAXE

### ByVaultName (Padrão)
```
Restore-AzKeyVaultCertificate [-VaultName] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByInputObject
```
Restore-AzKeyVaultCertificate [-InputObject] <PSKeyVault> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ByResourceId
```
Restore-AzKeyVaultCertificate [-ResourceId] <String> [-InputFile] <String>
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
O cmdlet **Restore-AzKeyVaultCertificate** cria um certificado no cofre de chaves especificado de um arquivo de backup.
Esse certificado é uma réplica do certificado de backup no arquivo de entrada e tem o mesmo nome do certificado original.
Se o cofre de chaves já contiver um certificado com o mesmo nome, esse cmdlet falhará em vez de sobrescrever o certificado original.
Se o backup contiver várias versões de um certificado, todas as versões serão restauradas.
O cofre de chaves em que você restaura o certificado pode ser diferente do cofre de chaves de onde você fez o backup do certificado.
No entanto, o cofre de chaves deve usar a mesma assinatura e estar em uma região do Azure na mesma geografia (por exemplo, América do Norte).
Consulte o Centro de Confiança do Microsoft Azure ( https://azure.microsoft.com/support/trust-center/) para o mapeamento de regiões do Azure para geografias.

SecretId      : https://mykeyvault.vault.azure.net:443/secrets/cert1/bd406f6d6b3a41a1a1c633494d8c3c3a

Este comando restaura um certificado, incluindo todas as suas versões, do arquivo de backup chamado Backup.blob para o cofre de chaves chamado MyKeyVault.
