# New-AzServiceFabricApplication

## SYNOPSIS
Crie um novo aplicativo de malha de serviço sob o grupo de recursos e cluster especificados.

## SINTAXE

### SkipAppTypeVersion (Padrão)
```
New-AzServiceFabricApplication [-ResourceGroupName] <String> [-ClusterName] <String>
 [-ApplicationTypeName] <String> [-ApplicationTypeVersion] <String> -Name <String>
 [-ApplicationParameter <Hashtable>] [-MinimumNodeCount <Int64>] [-MaximumNodeCount <Int64>] [-Force]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### CreateAppTypeVersion
```
New-AzServiceFabricApplication [-ResourceGroupName] <String> [-ClusterName] <String>
 [-ApplicationTypeName] <String> [-ApplicationTypeVersion] <String> -Name <String>
 [-ApplicationParameter <Hashtable>] -PackageUrl <String> [-MinimumNodeCount <Int64>]
 [-MaximumNodeCount <Int64>] [-Force] [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm]
 [<CommonParameters>]
```

## DESCRIPTION
Este cmdlet cria um novo aplicativo de malha de serviço sob o grupo de recursos especificado e o cluster. O parâmetro -PackageUrl pode ser usado para criar a versão de tipo, Se a versão de tipo já sair, mas estiver em estado 'Falha', o cmdlet perguntará se o usuário deseja recriar a versão de tipo. Se ele continuar no estado 'Falha', o comando interromperá o processo e lançará uma exceção.

## EXEMPLOS

### Exemplo 1
```powershell
PS C:\> New-AzServiceFabricApplication -ResourceGroupName "testRG" -ClusterName "testCluster" -ApplicationTypeName "TestAppType" -ApplicationTypeVersion "v1" -Name "testApp" -ApplicationParameter @{key0="value0";key1=$null;key2="value2"}
```

Este exemplo cria o aplicativo "testApp" em grupo de recursos "testRG" e cluster "testCluster". O tipo de aplicativo "TestAppType" versão "v1" já deve existir no cluster, e os parâmetros de aplicativo devem ser definidos no manifesto do aplicativo caso contrário, o cmdlet falhará.

### Exemplo 2: Especifique -PackageUrl para criar a versão do tipo de aplicativo antes de criar o aplicativo.
```powershell
PS C:\> New-AzServiceFabricApplication -ResourceGroupName "testRG" -ClusterName "testCluster" -ApplicationTypeName "TestAppType" -ApplicationTypeVersion "v1" -Name "testApp" -PackageUrl "https://sftestapp.blob.core.windows.net/sftestapp/testApp_1.0.sfpkg" -ApplicationParameter @{key0="value0";key1=$null;key2="value2"}
```

Este exemplo cria o tipo de aplicativo "TestAppType" versão "v1" usando a url do pacote fornecida. Depois disso, ele continuará o processo normal para criar o aplicativo. Se a versão do tipo de aplicativo já sair e o estado de provisionamento estiver em 'Falha' ele solicitará que decida se o usuário deseja recriar a versão de tipo.
