# Rename-AzContext

## SYNOPSIS
Renomeie um contexto do Azure.  Por padrão, os contextos são nomeados por conta de usuário e assinatura.

## SINTAXE

### RenameByInputObject (Padrão)
```
Rename-AzContext -InputObject <PSAzureContext> [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-TargetName] <String> [<CommonParameters>]
```

### RenameByName
```
Rename-AzContext [-Force] [-PassThru] [-Scope <ContextModificationScope>]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [-SourceName] <String> [-TargetName] <String>
 [<CommonParameters>]
```

## DESCRIPTION
Renomeie um contexto do Azure.  Por padrão, os contextos são nomeados por conta de usuário e assinatura.

## EXEMPLOS

### Exemplo 1: Renomear um contexto usando parâmetros nomeados
```powershell
PS C:\> Rename-AzContext -SourceName "[user1@contoso.org; 12345-6789-2345-3567890]" -TargetName "Work"
```

Renomeie o contexto para ' ' com a assinatura user1@contoso.org '12345-6789-2345-3567890' para 'Work'.  Após esse comando, você poderá direcionar o contexto usando 'Select-AzContext Work'.  Observe que você pode tabular os valores de 'SourceName' usando a conclusão da guia.

### Exemplo 2: Renomear um contexto usando parâmetros posicionais
```powershell
PS C:\> Rename-AzContext "My context" "Work"
```

Renomeie o contexto chamado "Meu contexto" para "Trabalho".  Após esse comando, você poderá direcionar o contexto usando o Select-AzContext Work
