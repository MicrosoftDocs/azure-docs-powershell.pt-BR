# Remove-AzIntegrationAccountReceivedIcn

## SYNOPSIS
Este cmdlet remove um número de controle de intercâmbio recebido específico por contrato e valor de número de controle.

## SINTAXE

```
Remove-AzIntegrationAccountReceivedIcn -ResourceGroupName <String> -Name <String> -AgreementName <String>
 -ControlNumberValue <String> [-AgreementType <String>] [-DefaultProfile <IAzureContextContainer>] [-WhatIf]
 [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
Esse cmdlet deve ser usado em cenários de recuperação de desastres para remover um número de controle de intercâmbio recebido da conta de integração para que o conector B2B possa processar novamente a mensagem quando a detecção de número duplicado estiver habilitada.
Em raras ocasiões, o número de controle de intercâmbio recebido pode ser reservado pouco antes de um desastre e antes que o conector B2B rejeite o intercâmbio como errôneo.
Em tais ocasiões, a operação pode querer permitir que o site de recuperação processe novamente o mesmo intercâmbio depois que sua carga for corrigida.
Forneça o parâmetro "-AgreementType" para especificar se os números de controle X12 ou Edifact devem retornar
