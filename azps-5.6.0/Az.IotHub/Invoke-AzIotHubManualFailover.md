# Invoke-AzIotHubManualFailover

## SYNOPSIS
Invocar o processo de failover para o Hub de IoT para a região de recuperação de desastres geo emparelhada.

## SINTAXE

### ResourceSet (Padrão)
```
Invoke-AzIotHubManualFailover [-ResourceGroupName] <String> [-Name] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### InputObjectSet
```
Invoke-AzIotHubManualFailover [-InputObject] <PSIotHub> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

### ResourceIdSet
```
Invoke-AzIotHubManualFailover [-ResourceId] <String> [-AsJob] [-PassThru]
 [-DefaultProfile <IAzureContextContainer>] [-WhatIf] [-Confirm] [<CommonParameters>]
```

## DESCRIPTION
Ele disparará o failover do hub de IoT para o local secundário. Essa ação causará perda de tempo e telemetria para sua solução. Esta é uma operação de longa duração e pode levar vários minutos para ser final. Tenha cuidado ao usá-lo.

## EXEMPLOS

### Exemplo 1
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub"
```

Iniciando o processo de failover do Hub de IoT "myiothub".

### Exemplo 2
```powershell
PS C:\> Invoke-AzIotHubManualFailover -ResourceGroupName "myresourcegroup" -Name "myiothub" -AsJob
```

Iniciando o processo de failover do Hub de IoT "myiothub" em segundo plano.
